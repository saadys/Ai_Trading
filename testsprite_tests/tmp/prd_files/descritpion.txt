Guide de Présentation du Projet Ai_Trading
Ce document fournit une vue d'ensemble complète du projet Ai_Trading, ses objectifs, son architecture technique et ses fonctionnalités clés. Il est conçu pour vous aider à présenter le projet de manière structurée et détaillée.

1. Vue d'Ensemble du Projet
Ai_Trading est une plateforme backend sophistiquée conçue pour l'analyse et l'exécution de stratégies de trading algorithmique. Elle combine l'analyse technique traditionnelle, le Machine Learning (LSTM) et l'analyse de sentiment (NLP) pour prendre des décisions de trading éclairées en temps réel sur les marchés de crypto-monnaies (principalement Binance).

L'objectif principal est de créer un système autonome capable de :

Collecter des données de marché en temps réel.
Analyser le sentiment du marché via les actualités.
Calculer des indicateurs techniques.
Exécuter des stratégies de trading basées sur ces signaux.
2. Architecture Technique
Le projet repose sur une architecture modulaire et asynchrone, favorisant la scalabilité et la performance.

2.1 Stack Technologique
Langage Principal : Python.
Framework Web : FastAPI (pour l'API REST et la gestion des flux).
Base de Données : PostgreSQL (avec SQLAlchemy et asyncpg pour l'accès asynchrone).
Broker de Messages : RabbitMQ (pour la communication entre les services et le traitement asynchrone).
Machine Learning : PyTorch / Transformers (pour FinBERT), Scikit-learn (pour les modèles classiques).
Conteneurisation : Docker (pour l'orchestration des services comme RabbitMQ et Postgres).
2.2 Flux de Données (Data Flow)
Market Data Collector : Connecté à l'API Binance via WebSocket, il reçoit les données de marché (Klines/Bougies) en temps réel.
Stream Processor : Traite les flux entrants bruts et les redirige vers le système de messagerie.
Queue Manager (RabbitMQ) : Agit comme une colonne vertébrale, distribuant les messages aux différents processeurs (Sentiment, Indicateurs, etc.).
Processors :
SentimentProcessor : Analyse les actualités financières pour déterminer le sentiment du marché (Positif, Négatif, Neutre) en utilisant le modèle FinBERT.
IndicatorService : Calcule des indicateurs techniques (RSI, MACD, moyennes mobiles, etc.).
Data Saver : Un worker dédié qui sauvegarde les données traitées et brutes dans PostgreSQL pour l'historique et l'analyse future.
3. Fonctionnalités Clés
3.1 Analyse de Sentiment (NLP)
Le projet intègre un module avancé d'analyse de sentiment (
SentimentProcessor
).

Modèle : Utilise FinBERT (Financial BERT), un modèle de langage pré-entraîné spécifiquement pour le texte financier.
Design Pattern : Utilise le pattern Strategy (SentimentStrategyFactory) pour permettre de changer facilement d'algorithme d'analyse (ex: FinBERT, Vader, TextBlob) sans modifier le cœur du système.
Flux : Les actualités sont consommées depuis une queue RabbitMQ, analysées, et le score de sentiment est republié pour être utilisé par les stratégies de trading.
3.2 Analyse Technique et ML
Indicateurs : Calcul temps réel d'indicateurs via IndicatorService.
Prédiction de Prix : Intégration de modèles LSTM (MLearning/LSTM_Model) pour prédire les mouvements futurs des prix basés sur les séquences historiques.
3.3 API et Streaming
L'API REST exposée par FastAPI permet de :

Démarrer/Arrêter les flux de données (/Root/v1/Streaming/start, /stop).
Obtenir le statut du système en temps réel.
Gérer les configurations de trading.
4. Structure du Code
Le code est organisé de manière professionnelle (Ai_Trading/app/) :

core/ : Configurations (Config.py), Logger, Gestion des exceptions.
models/ : Modèles de base de données (SQLAlchemy) et schémas Pydantic.
processors/ : Logique métier des différents processeurs (Sentiment, etc.).
routers/ : Endpoints de l'API.
services/ : Services utilitaires (QueueManager pour RabbitMQ).
Note : Le dossier Robot-Trading2 observé dans l'espace de travail est un environnement virtuel Python (venv) et ne contient pas de code source du projet.

5. Guide d'Installation Rapide
Pour présenter une démo ou installer le projet :

Pré-requis : Avoir Docker et Python installés.
Lancer l'Infrastructure : Démarrer RabbitMQ et PostgreSQL (souvent via un docker-compose up si disponible, ou manuellement).
Configurer : Créer un fichier .env avec les accès DB et RabbitMQ.
Démarrer l'Application :
uvicorn main:app --reload
Tester : Accéder à http://localhost:8000/docs pour voir l'interface Swagger de l'API.
